# ホリエモンAI学校 介護校｜AI活用例

**〜 介護現場の「困った」を、AIで解決する5つの方法 〜**

---

## はじめに

介護現場では、日々の記録業務・書類作成に多くの時間が費やされています。
ホリエモンAI学校 介護校では、**生成AI活用に関する240以上の動画講義**で、
現場スタッフが「明日から使える」AI活用スキルを身につけることができます。

ここでは、受講後すぐに実践できる代表的な活用例を5つご紹介します。

---

## 活用例 1｜介護記録の作成時間を半分に

### こんな課題はありませんか？

- 1人あたり毎日30分〜1時間、介護記録の作成に時間を取られている
- 夜勤明けのスタッフが疲労の中で記録を書いている
- ケアの合間に手が離せず、その場でメモを取れないことがある
- 記録の書き方にばらつきがあり、監査時に指摘を受けることがある

### AIでこう変わります

**Before**
```
利用者の様子を思い出しながら、一から文章を手入力。
1件あたり15〜20分。夜勤帯の記録はまとめて翌朝書くことも。
介助中はメモが取れず、後で思い出せないこともある。
書き方が人によって異なり、統一感がない。
```

**After**
```
ケアの直後にスマホで30秒〜1分、声で記録を残す。
「A様、朝食全量摂取、むせなし、機嫌よさそうでした」
——これだけでOK。AIが音声を文字に起こし、介護記録の
フォーマットに沿った文章を自動生成。1件あたり3〜5分で完了。
文体・用語が統一され、監査対応もスムーズに。
```

### 具体的なやり方

**方法A：音声入力で記録する（おすすめ）**

1. ケアの直後に**スマホの音声入力**で利用者の様子を話す（30秒〜1分）
2. 音声が自動で文字起こしされる
3. AIに「この内容を介護記録として整えてください」と指示する
4. フォーマットに沿った記録文が自動生成→確認して完成

**方法B：メモから記録する**

1. 利用者の状態を**箇条書きメモ**で残す
2. AIに「介護記録として整えてください」と指示する
3. バイタル情報・食事量・活動内容などを含む記録文が自動生成される
4. 内容を確認・微修正して完成

> **音声入力のメリット**: 手がふさがっている介助中や移動中でも記録を残せます。
> 記憶が新鮮なうちに話すので、情報の抜け漏れも防げます。

### AIへの指示文（プロンプト）の例

**音声入力からの記録作成**
```
以下は介護スタッフが音声入力で残した記録メモです。
介護記録のフォーマットに整えてください。
客観的な表現を使い、バイタル・食事・排泄・活動・特記事項の
項目に分けて記載してください。話し言葉は書き言葉に直してください。

音声メモ：
「A様、朝ごはん全部食べました、むせはなかったです。
血圧128の78、熱は36.4。午前中のレクに参加して塗り絵を
楽しそうにやってました。10時にトイレ誘導して排尿ありました。
お昼ご飯のあとに足が痛いって言ってたので看護師さんに伝えました。」
```

**箇条書きメモからの記録作成**
```
以下のメモを、介護記録のフォーマットで整えてください。
客観的な表現を使い、バイタル・食事・排泄・活動・特記事項の
項目に分けて記載してください。

メモ：
・朝食 全量摂取 むせなし
・BP 128/78 体温36.4
・午前レクに参加 塗り絵を楽しまれていた
・排泄 10時トイレ誘導 排尿あり
・昼食後「足が痛い」との訴えあり 看護師に報告済み
```

### 期待できる効果

| 指標 | Before | After |
|------|--------|-------|
| 記録作成時間（1件あたり） | 15〜20分 | 3〜5分（音声入力の場合） |
| 1日あたりの削減時間（10件の場合） | — | 約2〜3時間 |
| 記録のフォーマット統一率 | ばらつきあり | ほぼ100% |
| 介助中の記録 | 後でまとめて入力 | 音声でその場で記録可能 |

## 活用例 2｜ケアプランの自動作成

### こんな課題はありませんか？

- ケアプランの文章作成に時間がかかり、本来のアセスメントに集中できない
- アセスメント訪問の後、事務所に戻ってから改めて入力する二度手間が発生している
- 定型的な表現が多くなりがちで、利用者一人ひとりに合った計画が書きにくい
- 計画の見直し時期が重なると、対応が追いつかない

### AIでこう変わります

**Before**
```
アセスメント訪問でメモを取り、事務所に戻ってから
情報を整理してケアプランを手入力。1件あたり1〜2時間。
過去のプランをコピーして修正するため、内容が画一的になりがち。
訪問から時間が経つと、利用者の細かな様子を思い出せないことも。
```

**After**
```
アセスメント訪問中にスマホで音声録音。
利用者やご家族との会話、観察した内容を喋りながら記録するだけで、
AIがアセスメント情報を自動抽出→ケアプランを自動作成。
訪問先から事務所に戻る頃には、プランの原案が出来上がっている。
1件あたり15〜30分で完了。
```

### 具体的なやり方

1. アセスメント訪問中に**スマホで音声入力**しながら、利用者の状態・本人の意向・家族の意向・生活環境などを話す
2. AIが音声を文字起こしし、アセスメント項目（ADL・IADL・意向等）に自動分類
3. AIが「このアセスメントに基づいたケアプラン」を自動作成——長期目標・短期目標・サービス内容まで一気に生成
4. ケアマネジャーが内容を確認・最終調整して完成

> **ポイント**: アセスメントしながら喋るだけで、訪問→記録→プラン作成の工程が一つに。
> 「訪問して、戻って、入力して」の二度手間がなくなります。

### AIへの指示文（プロンプト）の例

**音声入力からケアプランを自動作成**
```
以下はアセスメント訪問中にケアマネジャーが音声入力した内容です。
この情報をもとに、ケアプランを作成してください。
長期目標・短期目標・サービス内容・頻度を具体的に記載してください。

音声メモ：
「B様、80歳女性、要介護2です。
左膝の変形性関節症があって、歩行には杖を使っています。
室内は手すりを使って何とか移動できるけど、外出はほぼできない状態。
ご本人は『できるだけこの家で暮らしたい、買い物にも行きたい』と
おっしゃっていました。
娘さんは『転倒が一番心配。週に何回かデイサービスに行ってほしい』と。
今のサービスは訪問介護が週2回と、福祉用具で杖と手すりを借りています。
浴室の段差が気になりました。手すりの追加を検討したほうがいいかも。
食事は自分で作れているけど、立ち仕事がつらそうでした。」
```

**既存のアセスメント情報からケアプランを作成**
```
以下のアセスメント情報に基づき、ケアプランを
作成してください。長期目標・短期目標・サービス内容を
具体的に記載してください。

【利用者情報】
・80歳女性 要介護2
・左膝の変形性関節症で歩行に杖が必要
・本人の意向：「できるだけ自宅で生活したい」
・家族の意向：「転倒が心配。デイサービスに通ってほしい」
・現在のサービス：訪問介護（週2回）、福祉用具貸与（杖・手すり）
```

### 期待できる効果

| 指標 | Before | After |
|------|--------|-------|
| ケアプラン作成時間（1件あたり） | 1〜2時間 | 15〜30分 |
| アセスメント→プラン完成まで | 訪問後、別日に作成 | 訪問当日中に完成 |
| 見直し対応のスピード | 1週間以上 | 1〜2日 |
| 文章の具体性 | 定型的 | 訪問時の生の情報が反映 |

## 活用例 3｜シフト作成の時間を大幅短縮

### こんな課題はありませんか？

- 毎月のシフト作成に管理者が丸1日〜2日かかっている
- 夜勤回数・連勤制限・資格配置など、考慮すべき条件が多すぎる
- 急な欠勤時の調整が大変で、管理者の負担が集中している

### AIでこう変わります

**Before**
```
スタッフの希望休・資格・労働時間制限をExcelや紙で確認しながら
パズルのように組み立てる。完成まで丸1日以上。
1人変更するとドミノ式に修正が必要になることも。
```

**After**
```
スタッフの条件（希望休・資格・夜勤上限等）をAIに渡すと、
配置基準を満たすシフト案を複数パターン提案。
所要時間は2〜3時間に短縮。急な変更にも素早く対応可能。
```

### 具体的なやり方

1. スタッフの情報（氏名・資格・勤務形態・希望休・夜勤上限等）をスプレッドシートにまとめる
2. 施設の配置基準（日勤帯の最低人数・夜勤の人数等）を条件として設定
3. AIに「この条件でシフト表を作成してください」と指示
4. 複数のシフト案が提案されるので、管理者が最終調整して確定

### AIへの指示文（プロンプト）の例

```
以下の条件で、来月の介護スタッフのシフト表を作成してください。

【施設条件】
・入所定員30名のユニット型特養
・日勤帯（7:00-16:00, 8:30-17:30）最低4名配置
・夜勤帯（16:30-翌9:30）最低2名配置
・介護福祉士を各シフトに1名以上配置

【スタッフ情報】
・Aさん：常勤、介護福祉士、夜勤月4回まで、3日と15日希望休
・Bさん：常勤、介護福祉士、夜勤月5回まで、希望休なし
・Cさん：常勤、初任者研修、夜勤月3回まで、毎週日曜休み希望
 ...（以下スタッフ情報を列記）

【制約条件】
・連続勤務は5日まで
・夜勤明けの翌日は休み
・週休2日を確保
```

### 期待できる効果

| 指標 | Before | After |
|------|--------|-------|
| シフト作成時間 | 丸1日〜2日 | 2〜3時間 |
| 条件漏れ・配置ミス | 月1〜2回発生 | ほぼゼロ |
| 急な変更への対応 | 数時間 | 30分〜1時間 |

## 活用例 4｜送迎表の自動作成

### こんな課題はありませんか？

- 毎日の送迎ルート・時間割の作成に、担当者が30分〜1時間かかっている
- 当日の欠席連絡やキャンセルが入るたびに、ルートを一から組み直す必要がある
- 利用者の住所・車いす対応・乗車時間の希望など、考慮すべき条件が多すぎる
- ベテランの「頭の中のルート」に依存していて、その人が休むと回らない

### AIでこう変わります

**Before**
```
利用者の住所一覧と地図を見ながら、手作業でルートを組む。
車いすの方の車両割り当て、乗車時間の希望、同乗の相性なども考慮。
毎朝30分〜1時間かけて送迎表を作成。
キャンセルが1件入ると、ルート全体の組み直しが発生。
ベテランドライバーの経験と勘に頼っている状態。
```

**After**
```
利用者リスト・条件をAIに渡すだけで、
最適な送迎ルート・車両割り当て・時間割を自動作成。
所要時間は5〜10分。
当日の欠席連絡にも「〇〇さんキャンセル」と伝えるだけで
ルートを即座に再計算。誰でも同じ品質の送迎表が作れる。
```

### 具体的なやり方

1. 利用者の情報（住所・車いすの有無・希望到着時間・特記事項等）をスプレッドシートにまとめる
2. 車両情報（台数・定員・車いす対応可否）と出発地点（施設の住所）を設定
3. AIに「この条件で送迎ルートと時間割を作成してください」と指示
4. ルート案が生成されるので、確認して確定。キャンセル時は該当者を伝えて再作成

> **ポイント**: 一度スプレッドシートを作れば、日々の作業は
> 「今日の利用者はこの人たち」と伝えるだけ。
> 急な変更にも数分で対応でき、ドライバーの属人化を解消できます。

### AIへの指示文（プロンプト）の例

**送迎表の作成**
```
以下の条件で、本日のデイサービス送迎表を作成してください。
効率的なルートを組み、各車両の出発時刻・到着予定時刻も記載してください。

【施設情報】
・施設住所：〇〇市△△町1-2-3
・デイサービス開始時間：9:30
・送迎車両：
  - 1号車：ハイエース（定員8名、車いす2台対応）
  - 2号車：軽ワゴン（定員4名、車いす不可）

【本日の送迎対象者】
・A様：〇〇市△△町5-6 車いす 9:15までに到着希望
・B様：〇〇市□□町2-3 自立歩行
・C様：〇〇市△△町8-1 自立歩行 同じエリアのA様と同乗希望
・D様：〇〇市◇◇町4-5 車いす
・E様：〇〇市□□町7-8 自立歩行 9:00以降のお迎え希望
・F様：〇〇市◇◇町1-2 杖使用

【考慮事項】
・車いすの方は1号車に配置
・乗車時間は1人あたり30分以内に抑える
・同じエリアの方はまとめて回る
```

**当日のキャンセル対応**
```
本日の送迎表からB様がキャンセルになりました。
ルートを再計算して、更新した送迎表を作成してください。
他の方の到着時間に影響が出る場合は、変更点を明示してください。
```

### 期待できる効果

| 指標 | Before | After |
|------|--------|-------|
| 送迎表の作成時間（毎日） | 30分〜1時間 | 5〜10分 |
| キャンセル時のルート変更 | 15〜30分 | 数分 |
| ベテラン不在時の対応 | 混乱・遅延が発生 | 誰でも同じ品質で作成可能 |
| 乗車時間の最適化 | 経験と勘に依存 | AIが最短ルートを計算 |

## 活用例 5｜NotebookLMで「いつでも聞ける研修」を作る

### こんな課題はありませんか？

- 新人が入るたびに同じ研修を繰り返している。教える側の負担が大きい
- マニュアルはあるが分厚くて読まれない。「どこに書いてあるか分からない」と聞かれる
- 夜勤中や休日に判断に迷っても、すぐに確認できる手段がない
- 外国人スタッフが増えているが、日本語のマニュアルだけでは理解しきれない

### NotebookLMとは？

Googleが提供する無料AIツールです。マニュアルや研修資料をアップロードすると、
**その資料の内容だけを参照して回答するAI**が作れます。
一般的なAIと違い、アップロードした資料以外の情報は使わないため、
**嘘の回答（ハルシネーション）を防げる**のが最大の特徴です。

さらに、以下の機能で研修コンテンツを自動作成できます：
- **Audio Overview**: 資料の内容をポッドキャスト風の音声に自動変換
- **スライド自動生成**: 研修用のプレゼン資料をPPTX形式で出力
- **質問応答**: スタッフがいつでも資料の内容について質問できる

### AIでこう変わります

**Before**
```
新人が入るたびに先輩スタッフが時間を取って研修を実施。
マニュアルは紙やPDFで配布するが、分厚くて読まれない。
夜勤中に「この場合どうすれば？」と迷っても確認する手段がない。
外国人スタッフはマニュアルの日本語が難しく、理解に時間がかかる。
```

**After**
```
マニュアル・研修資料をNotebookLMにアップロードするだけで、
「24時間いつでも質問できるAI研修講師」が完成。
夜勤中でもスマホで「転倒時の対応手順は？」と聞けば即回答。
Audio Overviewで通勤中に聞ける音声教材も自動生成。
スライドも自動作成されるので、研修の準備時間がほぼゼロに。
```

### 具体的なやり方

**ステップ1：NotebookLMに資料をアップロード（10分）**

1. NotebookLM（notebooklm.google.com）にアクセス
2. 新規ノートブックを作成
3. 以下の資料をアップロード：
   - 介護マニュアル（PDF）
   - 感染症対策ガイドライン
   - 事故・ヒヤリハット対応手順書
   - 就業規則・施設の運営ルール
   - 過去の研修スライド

**ステップ2：AI研修講師として活用（すぐに使える）**

- スタッフが知りたいことを質問するだけで、マニュアルに基づいた正確な回答が返ってくる
- 回答には**出典（どのマニュアルの何ページか）**が表示されるので、根拠が明確
- 夜勤中・休日でも24時間利用可能

**ステップ3：研修コンテンツを自動生成**

- **Audio Overview**: マニュアルの内容を15〜20分のポッドキャスト風音声に変換→スタッフが通勤中や休憩中に聞ける
- **スライド自動生成**: 研修テーマを指定するだけでPPTX形式のスライドが完成→そのまま研修で使える
- **要約・チェックリスト**: 長いマニュアルの要点だけを抽出したチェックリストを自動作成

### 活用シーンの例

**シーン1：夜勤中の緊急対応**
```
スタッフの質問：
「利用者が転倒して頭を打ちました。出血はないですが、
意識はあります。どう対応すればいいですか？」

NotebookLMの回答：
事故対応マニュアル（p.12-14）に基づき、以下の手順で対応してください。
1. 利用者を安静にし、バイタルサインを測定
2. 意識レベルの確認（JCS・GCSで評価）
3. 看護師（夜間オンコール：090-XXXX-XXXX）に連絡
4. 事故報告書を作成（様式は共有フォルダ内）
5. 翌朝、ご家族と主治医に報告
※頭部打撲の場合、24時間は経過観察が必要です。
（出典：事故対応マニュアル 第3章「転倒・転落時の対応」p.12-14）
```

**シーン2：新人スタッフの自主学習**
```
スタッフの質問：
「食事介助で気をつけるポイントを教えてください」

NotebookLMの回答：
研修資料「食事介助の基本」（p.5-8）より、主なポイントは以下の通りです。
1. 姿勢：座位を保ち、顎を少し引いた状態にする
2. ペース：一口の量はティースプーン1杯程度、飲み込みを確認してから次へ
3. 声かけ：「お魚ですよ」など、何を食べるか伝える
4. 観察：むせ・咳・表情の変化に注意
5. 記録：摂取量・むせの有無・所要時間を記録
（出典：研修資料「食事介助の基本」p.5-8）
```
