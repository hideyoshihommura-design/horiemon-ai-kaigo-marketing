PHASE 02

各ツールのアカウント開設＆ID取得

5つのツールを開設してIDを集めます。ここがいちばん時間がかかる工程です。

GA4
Google Analytics 4
サイトの全体動向を解析。広告に依存しない
取得するもの: 測定ID

  1. Googleアカウントでログイン

    analytics.google.com を開く

  2. 新規プロパティを作成

    「管理」（左下歯車）→「プロパティを作成」

    プロパティ名: 介護研修LP　タイムゾーン: 日本　通貨: 日本円

  3. 業種・ビジネス情報を入力

    業種: 教育　ビジネス規模: 中小企業

  4. データストリームを作成

    プラットフォーム: ウェブを選択

    ウェブサイトのURL: HubSpot公開ドメイン　ストリーム名: 介護研修LP - Web

  5. 測定IDをコピー

    表示される G-XXXXXXXXXX 形式のIDをコピーしてメモ

    後で使う: HTMLの G-XXXXXXXXXX プレースホルダー2箇所をこのIDで置き換えます

  6. 拡張計測機能をONに（既定でON）

    同じデータストリーム画面の「拡張計測機能」が有効になっていることを確認。スクロール・外部リンククリック・サイト内検索・動画視聴・ファイルDLが自動計測されます

Ads
Google Ads コンバージョン
Google検索・YouTube広告の効果を測定し、最適化AIに学習させる
取得するもの: 変換ID＋ラベル

  1. Google Adsにログイン

    ads.google.com を開く（広告アカウント未作成なら先に開設）

  2. コンバージョン管理画面へ

    右上「目標」→「コンバージョン」→「概要」

  3. 「+ 新しいコンバージョンアクション」をクリック

    種類: ウェブサイトを選択

  4. ドメインをスキャン or 手動設定

    「手動でコンバージョンアクションを作成」を選択

  5. コンバージョンの詳細を入力

    カテゴリ: 見込み顧客（リード）の発掘　名前: 介護研修_資料請求

    値: 任意（例: 5,000円相当）　カウント方法: 1回　計測期間: 30日

  6. タグの設定を選ぶ

    「Googleタグを自分で追加またはマネージャーを使用」を選択

    表示される AW-XXXXXXXXXX と XXXXXXXXX（ラベル）の2つをコピーしてメモ

    2つで1セット: 変換IDだけでなく、必ずラベルもセットで控える。サンクスページHTMLに両方入れます

取得するもの: Pixel ID

  1. Meta Business Suite にログイン

  3. 「ウェブ」を選択 → 「Meta Pixel」を選択

    発行された 16桁の数字（例: 1234567890123456）をメモ

    後で使う: HTMLの XXXXXXXXXXXXXXXX（16桁分）プレースホルダー2箇所をこのIDで置き換えます

  5. 設定方法を選択

    「コードを手動でインストール」を選択（HubSpotで管理しているため）

    表示されるコードはすでにHTMLに入っているのでコピー不要。確認だけしてスキップ

  6. Conversion API（CAPI）は今は未設定でOK

    後で設定すれば計測精度が上がりますが、初回はPixelだけで十分

PHASE 03

取得したIDをHTMLに差し替える

サンクスページHTMLの「XXXXX」プレースホルダーを実際のIDで埋めます。GA4/Google Ads/Meta Pixel/LINEの4種類です。

差し替えイメージ

サンクスページHTML（thanks-kaigo-subsidy-acg.html）の以下のような行を変換します。

Before（プレースホルダー）
fbq('init', 'XXXXXXXXXXXXXXXX');
fbq('track', 'Lead', {
  content_name: 'kaigo-subsidy-document',
  currency: 'JPY',
  value: 1.0
});
→
After（実IDに置換）
fbq('init', '1234567890123456');
fbq('track', 'Lead', {
  content_name: 'kaigo-subsidy-document',
  currency: 'JPY',
  value: 1.0
});

同じ要領で:

  • 📌 G-XXXXXXXXXX → GA4測定ID（2箇所）
  • 📌 AW-XXXXXXXXXX → Google Ads変換ID（2箇所）
  • 📌 AW-XXXXXXXXXX/XXXXXXXXX → 変換ID+ラベル（1箇所、スラッシュ区切り）
  • 📌 XXXXXXXXXXXXXXXX（16桁） → Meta Pixel ID（2箇所）
  • 📌 XXXXXXXXXX（10桁） → LINEタグID（複数箇所）
HubSpot反映の手順: 差し替えたHTMLをHubSpot Design Manager > Coded files の該当ファイルに貼り直す → 公開 / Update

📍 LP側にも追加で入れるもの

サンクスページにはタグが入っていますが、LP側はまだ素のHTMLです。LP側に最低限以下を追加することで、リターゲ広告や精度の高い最適化が可能になります:

  • ●　GA4測定タグ（PageView計測 ＋ 拡張計測の発火元）
  • ●　Meta Pixel base code（PageView ＋ ViewContent。CV発火は無し）
  • ●　LINE Tag base code（PageView。CV発火は無し）
  • ●　Microsoft Clarity tracking code（必須）
原則: LPには「閲覧計測」、サンクスページには「CV計測」を入れる。役割分担です。
PHASE 04

動作確認：テストイベントを打つ

タグを入れた直後は必ずテスト。自分のスマホ・PCで実際にLP→フォーム送信→サンクスページの流れを通してみて、各社の管理画面で受信を確認します。

GA4

📍 確認場所:
左メニュー「レポート」→「リアルタイム」
⚡ 見るべき指標:
過去30分のアクティブユーザーが1（自分）になっているか、page_locationが期待のURLになっているか。generate_lead イベントが発火しているか

Google Ads

📍 確認場所:
「目標」→「コンバージョン」→該当アクション
⚡ 見るべき指標:
「タグの確認」セクションで「最近のアクション: 検出されました」と表示されればOK。最大24時間ラグあり

Meta Pixel

📍 確認場所:
Events Manager→該当Pixel→「テストイベント」タブ
⚡ 見るべき指標:
「テスト用ブラウザ」でURLを開くと、その場で PageView Lead イベントの受信がリアルタイム表示される

LINE Tag

📍 確認場所:
「共有ライブラリ」→「LINEタグ」→該当タグ
⚡ 見るべき指標:
「タグ送信履歴」「テストモード」でPV/CVが受信されているか確認

Clarity

📍 確認場所:
プロジェクトダッシュボード（インストール後すぐ）
⚡ 見るべき指標:
右上に「インストール完了」表示。翌日にはヒートマップとセッション録画が自動で溜まり始める
ブラウザ拡張で便利チェック: Chromeに「Meta Pixel Helper」「Tag Assistant Legacy（Google Ads/GA4）」を入れると、ページを開いた瞬間に「どのタグが発火したか」が拡張機能アイコンで一目で確認できます
PHASE 05

計測したCVを広告の目標として登録する

CVが計測できるだけでは不十分。各広告マネージャに「これが目標」と教え込むことで、AIが「同じような人を連れて来る」最適化を始めます。

Ads
Google Ads キャンペーン設定
作成したコンバージョンを「主要コンバージョン」として登録

  1. キャンペーン作成時に「コンバージョンを取得」を選択

    「リード獲得」または「ウェブサイトのトラフィック」目標で開始

  2. コンバージョン目標を選択

    先に作成した「介護研修_資料請求」を「アカウントのデフォルトとして使用」にチェック

  3. 入札戦略を選ぶ

    初期: コンバージョン数の最大化　CVが50件/月貯まったら: 目標CPA に切り替え

Meta
Meta広告マネージャ設定
PixelのLeadイベントを最適化目標として指定

  1. キャンペーン目的: 「リード」を選択

    広告セットレベルで「コンバージョン場所: ウェブサイト」を選択

  2. Pixel & コンバージョンイベント選択

    該当のMeta Pixelを選択　パフォーマンスゴール: 「Leadの数を最大化」

  3. カスタムコンバージョン（より精密に）

    Events Manager→「カスタムコンバージョン」→新規作成

    ルール: URL に thanks-document-request を含む　標準イベント: Lead　名前: 介護研修_資料請求CV

    これを広告セットの最適化対象にすると、より正確

LINE
LINE広告マネージャ設定
作成したCVをキャンペーンの最適化目標に設定

  1. キャンペーン目的: 「ウェブサイトコンバージョン」

    新規キャンペーン作成時に選択

  2. コンバージョン地点を選択

    先に作成した「介護研修_資料請求」CVを選択

  3. 入札方法を選ぶ

    初期: 自動入札（コンバージョン数最大化）　CVデータが溜まったら目標CPAに移行

配信開始後は「数字を見て、判断し、施策に反映する」サイクルを回します。頻度ごとに見るべき指標を分けて運用すると迷子になりません。

毎日 5分

異常検知のみ

  • 広告が止まっていないか（配信ステータス）
  • CPAが前日の3倍など異常値出ていないか
  • 予算上限到達していないか
毎週 30分

パフォーマンスレビュー

  • CV数・CPA・CTR・CVR の推移
  • チャネル別の効率比較
  • クリエイティブ別の成果
  • 勝ち広告に予算寄せ、負け広告停止
隔週 1時間

LP改善のヒント収集

  • Clarityでヒートマップ確認
  • 離脱多発ポイントを特定
  • セッション録画で5本見る
  • 仮説→次のABテスト計画

戦略レビュー

  • ROI（受講料売上 ÷ 広告費）算出

TROUBLESHOOTING

よくあるトラブルと対処法

初回セットアップ後によく起きる問題と、確認すべきポイントを整理しました。

CVが1件も計測されない
① HTMLのID差し替え漏れ — ブラウザでサンクスページを開きF12で「コンソール」を確認、scriptエラーがないか
② HubSpot反映漏れ — Design Managerでコード更新後「公開」または「Update」ボタンを押し忘れていないか
③ URLが違う — 各社CV定義のURLとサンクスページの実URLが完全一致しているか
各社の管理画面でCV数が違う
これは 正常な動作です。アトリビューション（手柄配分）のルールが各社で違うため、同じ1件のCVを複数社が「自社経由」と主張します。
→真の総CV数は HubSpot側のフォーム送信数で見るのが正解。広告側のCV数は「最適化AIへの学習データ」と割り切る
サンクスページに飛ばずインライン表示になる
HubSpotフォーム設定の「送信時」が 「段落」 のまま。「リダイレクト」 に切替えてURLを指定する必要あり
・場所: フォーム編集画面→「コンテンツ」→「送信時」プルダウン
iPhoneユーザーからのCVが少ない
iOS 14以降の ATT（App Tracking Transparency）でブラウザ追跡が制限されているため、特にMeta広告でiOSユーザーのCV検知率が下がります。
→対策: Conversions API（CAPI） の導入。HubSpotにはMeta連携機能があるので、これでブラウザCookie経由でなくサーバー間連携でCV送信できる
Clarityでヒートマップが見られない
① データが溜まっていない — 設置から24時間以上、最低100セッション必要
② タグの設置場所が間違い — <body>内ではなく <head>内に設置されているか確認
③ プロジェクトURLとアクセス先URLが違う — Clarity側で設定したサイトURLと、実際にアクセスしているURLが一致しているか

セットアップ完了チェックリスト

すべてにチェックが入れば、計測の土台は完成です

  • Phase 2-2. Google Ads変換ID + ラベルを取得した
