01

勤務表の自動生成

課題：1日かかっていたシフト作成。月末はいつも残業

職員リスト・勤務ルール・希望休の3ファイルを渡し、日本語で「来月の勤務表を作って」と依頼。早番・日勤・遅番・夜勤・休みの記号配置から、有資格者の配置確認まで自動で。

👤「来月のシフトを作成してください。希望休は最優先で。夜勤回数は公平に配分してください。」
🤖「職員リストを読み込みました。ルールを解析しました。組み合わせを計算中…完成しました。早番8名/日勤10名/夜勤2名で配置、介護福祉士は各シフト1名以上を確保しています。」
02

集計表の自動作成

課題：夜勤の偏り・希望休の反映状況を電卓で追っていた

「集計も付けて」の一言で、職員ごとの勤務日数・夜勤回数・希望休の反映状況を自動で集計。公平性のチェックが瞬時に終わる。

👤「各職員の勤務日数と夜勤回数の集計も付けてください」
🤖「集計表を作成しました。Aさん 勤務22日・夜勤4回・希望休3/3反映、Bさん 勤務21日・夜勤4回・希望休2/2反映…（以下20名分）。夜勤回数は3〜5回の範囲で均等に配分されています。」
03

急な変更への即時対応

課題：1人体調不良が出るたび、シフト全体を組み直していた

「Cさんを今週夜勤から外して。代わりをルール内で再配置して」と話し言葉で伝えるだけ。再生成は数秒で完了。

👤「Cさんが体調不良で今週夜勤に入れません。再調整してください」
🤖「Cさんの今週の夜勤2回をDさん・Eさんに振り替えました。連勤上限・夜勤明け休みのルールも維持しています。差分を表示します。」
04

暗黙知の見える化

課題：ベテラン担当者しかシフトが組めず、引き継ぎが進まない

Claudeに使うために「頭の中のルール」を文章にすることが、そのまま業務の見える化に。引き継ぎが格段に楽になる副次効果も。

📝 ルールファイルに書き出した例
「日勤は8名以上配置 / 夜勤は2名体制 / 介護福祉士を各シフトに最低1名 / 連勤は5日まで / 夜勤明けの翌日は休み / 新人は3ヶ月間ベテランとペア配置」… 暗黙知が、誰でも読める形に。

{% module "faq_content" path="@hubspot/rich_text", label="FAQ 本文", html="
パソコンが苦手な職員でも使えますか？
Claude Codeは日本語で指示が出せるので、特別なプログラミング知識は不要です。最初の設定だけ少しハードルがありますが、私たちのAI学校でも導入支援は手厚くしています。「絶対無理」とおっしゃっていた方が、1ヶ月後には自分で触っているケースもたくさんあります。
個人情報を扱う上での注意点は？
Claude（有料プラン）は入力データを学習に使わない設定がデフォルトです。それでも、まずは職員さんを匿名化（「職員A、B、C」）した状態で運用を始めることを強くおすすめしています。施設の情報管理規定との照合も大切。ウェビナーで具体的な手順をお伝えします。
小規模事業所ですが、コストは見合いますか？
月数千円のコストで、シフト作成の時間を大幅に削減できれば、人件費換算で十分元が取れるケースが多いです。むしろ、人手の少ない小規模事業所こそ効果が大きい。担当者の負担減が施設全体に響きます。
既存のシフト作成ソフトとの違いは？
専用ソフトは決まった操作に沿って使うのが基本です。Claude Codeは日本語で柔軟に指示できるのが大きな違い。特に介護現場の「例外対応」への強さが特徴です。両方を使い分けている施設もあります。
AIが変なシフトを作ったらどうしますか？
Claude Codeはあくまで「下書きを作るツール」です。出力された勤務表は必ず人間が最終チェックしてから運用するので、AIの間違いがそのまま現場に出ることはありません。最終判断は人間、というルールは絶対に守ってください、と必ずお伝えしています。
導入までどのくらいかかりますか？
アカウント作成と初期設定で1日。最初のシフトを試しに作るまで1週間。本番運用までは1〜2ヶ月が目安です。私たちのAI学校では、この期間を伴走しながら一緒に進めるサポートもしています。
当日参加できない場合はどうなりますか？
録画を後日配信します。お申し込みいただければ、参加特典（スライド資料・プロンプト集・チェックリスト）もお受け取りいただけます。
